После получение конца данных, выполняется поиск:

	2. Конечный индекс блока заголовков
Функция FindNextHeaderField вызывается в цикле, каждый вызов должен найти либо 1 либо 2 случай.

Поиск происходит в данных, которые уже сохранены в памяти клиента.

Если функция возвращает -1, значит в буфере не было <CRLF><CRLF>, а значит не было блока заголовков, а значит данные не корректные.

У меня после первого запуска функции по логике предыдущими символами явл CRLF, так как если функцию запустилась снова, значит в прошлой итерации был найден очередной заголовок. А если след символ является CR('\r'), значит унас ситуация конца блока данных. Если не применить это условие, то при анализе след заголовка, в случае появление первыми символами CRLF, программа не сможет опознать конец блока заголовков, так как я не сохраняю нигде состояние прошлого запуска функции ( решил не нагружать доп переменными), по этому просто считаю, что если осуществлен вход в функцию, значит до этого был CRLF.
В случае первого запуска функции, когда в буфере будет стоять первыми символами CRLF, функция выдаст, с учетом предположения что до этого был CRLF, результат нахождения конца блока данных. для добавления адресов. То есть FieldToCcBcc выполняет для этих заголовков одно и тоже, кроме добавления адресов в массив.
Получает: указатель на расфальцованную строку значения заголовка, длину и структуру, где хранятся какие типы адресов в строке,
объект класса, в котором хранятся адреса и указатель на метод, который добавляет адреса для конкретного заголовка.
Возвращает true - ок, false - ошибка


						DateTime
Функция анализирует заголовки содержащие date-time.
В нее входит функция проверки есть ли День недели или сразу Дата - CheckDayOfWeek. 
Возвращает 1 - если есть, 0 - нет, -1 ошибка. В том числе проверяет есть ли вообще символы в строке кроме CRLF.
Если есть день недели, записывает в переменную-ссылку start_day_of_week позицию начала дня недели.

Переменная remaining_len хранит в себе остаток длины от исходной. Используетсяпри анализе, по ней перед 
проверкой очередного значения(месяца, дня, года, число и тд) проверяем, есть ли такое кол-во символов, 
которое требует значение.
Функция ProcessingDayOfWeek. Проверяет корректность дня недели+запятая.
Функция SearchNextNonWhitespaceChar используется почти в каждой функции обрабатывающая date-time. Она ищет первый не пробельный символ.
Функция ProcessingDayMonthYear и TimeOfDay. Анализ значений проход последовательно, так как известно, что должно быть в этом типе заголовке.

						IdentificationFields
Функция выполняется для заголовков, содержащих объект msg-id (поле идентификации).
Она проверяет корректное использование структуры <addr> ... <addr> \r\n. Не проверяет наличие символов в <>.
Проверяет наличие символа "(qouted_string), [ и ](domain literal). Они запрещены.
Получает: указатель на значение заголовка и структура с типами адресов.
Возвращает: true - успех, false - ошибка

						Общее
При получении всех данных от команды data в конце данных устанавливается '\0' и общее кол-во данных.
После расфальцовки значения заголовка создается новый буфер, который содержит расфальцованное значение + CRLF + '\0'.
	Такой буфер содержит первым символос НЕ пробельный символ, даже если в оригинальном значении заголовка он начинался 
	с пробела.
При выполнении функции анализа значения заголовка происходит:
	Сохранение unfolding_str в переменную связанную с заголовков
	В массив хранение начала адресов сохраняется адреса начала каждого адреса из сохраненнного unfolding_str
	То есть сохраняется строка всех адресов и потом берется отдельно из этой строки, которые означают начала 
	конкретных адресов



Примечание:
Для DateTime не проверяется строгое соответствие кол-ва дней конкретному месяцу. Проверяется только не больше 31.
Так как клиент может записать любое число и дату не нужно проверять строго. Проверяют в другом заголовке (received).

Не во всех типах заголовка соблюдены требования RFC 5322. 
	В заголовках с заданием даты и времени в конце RFC разрешает комментарии, у меня можно только пробельные символы.
	В объекте name_addr после <> можно комметарии. У меня только пробельные символы.
